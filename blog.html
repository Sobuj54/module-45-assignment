1. What is the difference between map() and forEach() and filter() and find() ?

Ans: The difference between map() and forEach():
map():
i. The map() method is used to transform the elements of an array.
ii. The map() method can be used with other array methods, such as the filter() method.
iii. The map() method returns a new array with the transformed elements.
forEach():
i. The forEach() method is used to loop through the elements of an array.
ii. The forEach() method cannot be used with other array methods.
iii. The forEach() method returns undefined value.
The difference between filter() and find():
filter():
i. The filter() method is used to filter all the elements.
ii. The filter() method returns the element that matches and removes the element that does not match.
iii. The filter() method searches through all the elements.
iv. The filter() method does not change the original array.
v. The filter() method does not execute the function for empty elements.
vi. In filter() method a value is passed to the function as this value.
find():
i. The find() method is used to find all the descendant elements of the selected element.
ii. The find() method finds the element in the DOM tree by traversing through the root to the leaf.
iii. The find() method searches through all the child elements only.
iv. It does not execute the function for empty elements. v. It does not change the original array.
vi. This method returns undefined if no elements are found.

2. What is the difference between Local Storage and Session Storage and Cookies ?

Ans:
Local Storage:
The storage capacity of local storage is 5MB/10MB. As it is not session-based, it must be deleted via javascript or manually. The client can only read local storage.There is no transfer of data to the server.There are fewer old browsers that support it
Local storage has 4 methods:
i. localStorage.getItem(key)
ii. localStorage.removeItem(key)
iii. localStorage.clear()
iv. localStorage.setItem(key, value)
Session Storage:
The storage capacity of session storage is 5MB. It’s session-based and works per window or tab. This means that data is stored only for the duration of a session, i.e., until the browser (or tab) is closed. The client can only read local storage. There is no transfer of data to the server. There are fewer old browsers that support it
Session Storage has 4 methods:
i. sessionStorage.setItem(key, value)
ii. sessionStorage.getItem(key)
iii. sessionStorage.removeItem(key)
iv. sessionStorage.clear()
Cookies:
The storage capacity of Cookies is 4KB. Cookies expire based on the setting and working per tab and window. Both clients and servers can read and write the cookies. Data transfer to the server is exist. It is supported by all the browser including older browser.
